How to use this dashboard

1. Cloud Sync — MongoDB Atlas (แนะนำ)

กดปุ่ม ⚙ MongoDB → ใส่ API URL ของ server → Test → บันทึก → ข้อมูลจะ sync อัตโนมัติทุกครั้งที่แก้ไข

ถ้าต้องการ backup: ใช้ Export JSON เพื่อ download ไฟล์สำรอง หรือ Import JSON เพื่อ restore

2. Add, Edit, Delete projects
  • กด + New Project ที่มุมขวาบน เพื่อเพิ่ม
  • คลิกที่แถวใดก็ได้ในตาราง Project List เพื่อแก้
  • ใน Kanban ลากการ์ดข้าม column เพื่อเปลี่ยน Status
  • กด Edit/Delete ในตารางเพื่อจัดการ
3. Comments

เปิด project เพื่อแก้ → ที่ส่วน "Comments" สามารถเพิ่ม note สถานะงาน, update จากลูกค้า, ฯลฯ — comment ถูกเก็บใน JSON file

4. Alerts
  • 🔴 Overload — Owner ที่มี peak ≥ 7 concurrent
  • 🟡 Deadline ใกล้ — End date เหลือ < 7 วัน
  • 🟡 Stale brief — รับ brief แล้ว > 14 วันแต่ยังไม่มี Launch date
5. Workload views
  • Individual — แต่ละคน (Owner + Support)
  • Department — โหลดรวมรายแผนก (Account Management, BD, ฯลฯ)
  • By Function — Roles ที่ถูก request

Project Intake & Resource Allocation

Capacity rule: Owner ไม่ควรมี peak ≥ 7 — ถ้าเกินต้องเพิ่ม Support หรือ defer
Pitching cap: Owner ที่ peak ≥ 5 ในเดือนใด ห้าม assign Pitching ใหม่ในเดือนนั้น
Role pre-booking: Strategy, Creative, Designer, KOL = high demand — จองล่วงหน้าใน calendar ก่อน confirm Launch

☁️ Cloud Sync — MongoDB Atlas Settings

วิธีตั้งค่า (ทำครั้งเดียว)

  1. Upload โฟลเดอร์ api-server/ ขึ้น GitHub (repo ใหม่หรือ repo เดิมก็ได้)
  2. เปิด render.com → New → Web Service → เลือก repo นั้น
  3. ตั้งค่า Environment Variables ใน Render:
    MONGO_URI = connection string จาก Atlas (Connect → Drivers)
    API_SECRET = รหัสที่คุณกำหนดเอง (ถ้าต้องการ)
  4. Deploy → Copy URL ของ service (เช่น https://my-api.onrender.com)
  5. กรอก URL นั้นด้านล่าง แล้วกด Test → บันทึก
